# 整理 wiki 结构（organize-wiki 变体）

> **轻量纯文档分支**：重排、扩张 wiki 的**层级与结构**（区别于 [fix-wiki-issues 变体](fix-wiki-issues.md) 修内容正确性）。**不走三段式 Workflow、不发 subagent、不动代码、无需 spec**——直接按下面流程做。

## 流程

1. **通读** `docs/wiki/`（六区：01 业务 / 02 概念 / 03 架构 / 04 设计 / 05 ADR / 06 里程碑与问题）。
2. **诊断**：找冗余、过期、交叉错位、层级失衡（一页塞多职责 / 该拆未拆 / 目录与内容不符 / README 索引过期）。
3. **重组**：拆页或并页、调目录结构、扩张子页层级、补/修各区 README 索引。
4. **守三硬规矩**：
   - **单向推导**：下游页只引上游页（架构引业务，不反向）。
   - **单源**：一事一处权威，别在多处复制；要么链接、要么沉淀。
   - **每页一职责**：每页开头一句「本页职责：回答什么问题」；一句话说不清就拆页。
5. **修链接与计数**：重组后所有相对链接可达、页内计数/状态与实际一致。
6. **收尾：提交 + 合并 main（不 push）**：落盘即闭环——开分支提交（`docs(wiki): ...` scoped 前缀）、切 main `git merge --ff-only <分支>`、删分支。**不问用户、不 push**；git 命令一律 `--no-pager`。

## 边界

- 只动**结构/层级/索引**；内容正确性问题（链断、设计-实现漂移）归 [fix-wiki-issues 变体](fix-wiki-issues.md)。
- `里程碑.md` 人工维护，**不改**。
- 不动代码，无验收测试；自查 = 链接可达 + 单源无重复 + 每页职责清晰。
