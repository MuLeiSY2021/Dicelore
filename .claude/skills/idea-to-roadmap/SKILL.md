# 点子归位 + 一路下沉（idea-to-roadmap）

**点子有海拔。** 用户抛来的一句话可能是一个宏大愿景，也可能是一个具体 bug。本 skill 的活：先把每条点子放到它**真正的海拔**（最高层是里程碑里的宏大目标/愿景），**再顺着海拔一路下沉**到可执行的 backlog 票 + 路线图批次。先归里程碑、再下沉——别一上来就当成普通 ticket 塞进 backlog，那样会丢掉「这想法到底服务哪个大目标」的脉络。

轻量、不发 subagent、不动代码。

## 输入
用户抛来的一堆散点子/想法，海拔不一：可能是「我想让框架将来能 X」这种愿景，也可能是「这里有个 bug」这种零碎 fix，混杂前后端/core。

## 流程

### 1. 先归里程碑海拔（入口，别跳过）
对**每条**点子先问一句：它服务 / 扩张哪个里程碑（宏大目标·愿景）？据答案分流：

- **全新的宏大目标/愿景** → 在 [`里程碑.md`](../../../docs/wiki/06-里程碑与问题/里程碑.md) 追加一个新里程碑块，按页内既定格式「宏大目标 → 拆成块 → 怎么实现」写，块内项标 ⬜。
- **扩张/细化已有里程碑** → 在对应里程碑块下补一条 ⬜ 子目标/块。
- **纯属某里程碑内的零碎 fix** → 不必动里程碑正文，但**记清它归属哪个里程碑**（下游 backlog 条目要链接回去，留 traceability）。

**写权边界（要紧）**：里程碑.md 的铁律是「AI 不得在**无人干预**下自行改动」。用户调本 skill 抛点子**本身就是人工干预**，所以 skill 内**可以**写里程碑——但只追加/细化**未来目标块（⬜）与「怎么实现」**；**不擅自标 ✅ 完成、不改写已达成的历史**（完成态由人在真达成那一刻定）。

### 2. 下沉到 backlog 三池
落 [`backlog-<层>.md`](../../../docs/wiki/06-里程碑与问题/)。把上一步的里程碑块拆成具体 fix/feat，逐条判定 `{层, 类型}` 落对应池：

- 前端 = `apps/web`（组件/渲染/路由/i18n/视觉）
- 后端 = `apps/orchestrator`（HTTP·WS 接口/会话生命周期/编排）
- core = `packages/core`(+`shared`)（引擎/数据层/MCP/gm-core/团本构建/eval）

补全字段：`类型(fix|feat)·来源·是否随规模恶化·所属主题·所属里程碑·下一步/依赖`。
**去重聚类**：同一架构病的多个症状归到一个**主题**下（N 个 ticket 常是 1 个决策）；跟既有条目比对，别重开。

### 3. 下沉到路线图
落 [`路线图.md`](../../../docs/wiki/06-里程碑与问题/路线图.md)。把池里的活编进/重排批次：**反复出现 + 随规模恶化 = 最高优先级**，进靠前批次。每批列「修哪些 fix / 提哪些 feat」，链接回三池条目 + 里程碑。

> **一路沉到底**：每条点子都要走完里程碑 → backlog → 路线图，别停在里程碑层。万一某愿景太大、现在无法负责任地拆细，也至少在 backlog 留一个粗主题、在路线图留一个「未来批次」stub，让它有去处、不在里程碑层断头。

### 4. 收尾：提交 + 合并 main（不 push）
落盘即闭环——开分支提交（`git checkout -b docs/idea-<主题>`，message 用 `docs(idea): ...` 或 `docs(backlog): ...` scoped 前缀），切回 main `git merge --ff-only <分支>`、删分支。**不向用户问"要提交吗"**，也**不 push**（push 由用户单独指令）。git 命令一律 `--no-pager`。

## 硬约束
- **海拔优先**：先归里程碑、再一路下沉到 backlog + 路线图，一次到底；别把宏大愿景直接当普通 ticket 塞 backlog。
- **里程碑.md**：AI 不得在**无人干预**下自行改动；本 skill 由用户抛点子触发即干预，故可在其内追加/细化未来目标块（⬜）与「怎么实现」，但**不标 ✅、不改写已达成历史**。
- **路线图.md**：AI 维护、可重排。
- 只做归位/拆解/聚类/排序/编批，**不直接动手实现**——实现走 `roadmap-delivery-workflow`。
- **落盘即提交合并 main**：不停下来问用户；不 push。
- git 命令一律 `--no-pager`（否则 less 卡死 Bash 会话）。
