# 第零步参考 · 状态机建模法 + Dicelore 现有模型

> 状态机是**总体设计的根**：接口是"有哪些门"（静态），状态机是"门怎么串成合法行为"（动态）。先把动态图景定对，第一步的接口只是它每条转移的触发器，第二步的测试只是遍历它。
> **live 模型看最新一轮**：Dicelore 的实际状态机（含全部 mermaid 图）在最新一轮 `docs/tdd/acceptance-loop-<date>/README.md` §1。**每轮从它出发精修，别从零重画**——重画必与既有模型漂移。本文件只定**建模法与命名骨架**。

## 两类状态（正交，别混）

- **实体状态**：后端持有的**持久实体**的存续态（会话、团本产物库）。这是本 skill 主要验的。
- **页状态**：每个页面一台 UI 状态机；**页状态 = 实体状态的投影 + 页自身 UI 状态**（如哪个面板开着，与会话是否活跃无关）。
- **导航状态 = 导航页（壳）的页状态**，不是独立第三轴。

映射：一个页 DISPLAYS 一个实体；两轴正交，但同属一个功能整体，都要建。

## A. 实体状态机

- **A1 会话生命周期**：`无 / 活跃 / 空 / 归档`，dicegm 与 loregm **共享此骨架**（映射后端共享的 `Session`/`TurnResult` 抽象）。
  - 无→活跃（创建）；活跃→无（删除）；活跃→活跃（drive-turn / 部分 rewind）；活跃→**空**（rewind 到开头 = empty）；活跃→**归档**（域内到终态·留档）；空/归档→活跃（继续）。
  - 关键：会话存续态与"内部玩到哪/建到哪"**解耦**。域内的 game_end / commit 不是会话终态，只**触发** `活跃→归档`。
- **域机（活跃内嵌，两内根）**：
  - **dicegm 跑团回合循环**：未开场→(kickoff)→待输入 →(玩家自由文本)→ 生成中 →待输入；含明骰闸子机（待掷: awaiting→committed）、待选、终局。
  - **loregm 自由编排**：待输入 →(**作者**自由文本)→ 编排中 →待输入（回合内 loregm 自主调 build MCP，顺序自定）；素材上传/搜索可选；作者觉得够了→校验→提交。
  - 两内根**回合层同构**：`待输入 →(自由文本 drive-turn)→ 处理中 → 待输入`，差异只在"回合内产什么"（叙事 vs build 工具调用）。阶段（世界观/NPC/…）对 loregm 是 guideline 非硬序。
- **A4 catalog 产物库**（独立于任何 session）：无版本→有草稿→已提交版本→(tag)；loregm 写、dicegm import 读（默认取 head）。

## B. 页状态机（各页一台）

导航页(壳) · 主页 · 团本目录页 · 跑团页 · 团本制作页 · 配置页。要点：

- **主页**：指南为主（怎么用 + 使用手册在哪）+ **最近一个会话摘要**，不投影全量列表。
- **团本目录页**：跑团入口——开始游戏（选版本·默认最新）→建 dicegm 会话+import→跳跑团页；可导入团本。
- **跑团页 / 制作页**：顶栏 **session bar**（滚动列所有会话：最近活动日期/所属团本/最新回复内容）；页体投影对应域机（A2/A3）+ 面板/编辑/校验等纯 UI。跑团页直接进且无活动会话 → 提示先导入/创建。
- **配置页**：七子页（通用/服务与网络/MCP服务器/模型连接/主题外观/数据与存储/关于），机制统一 = 表单 + 持久化 + 连接测试三态（测试中/成功/失败）。

## Mermaid 约定

- 用 `stateDiagram-v2`；复合状态表达"活跃内嵌域机""续玩层内嵌回合循环"。
- 转移标注触发（端点/事件）；红点边标 `❌/⚠️`（缺/存疑/超前）。
- 域内终态用 note 标"⇒ 会话 活跃→归档"，别在会话层重复画终态。
