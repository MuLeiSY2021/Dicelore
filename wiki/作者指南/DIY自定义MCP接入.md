# DIY 自定义 MCP 接入

> **本页职责**：怎么给你的团本接一个外部 MCP server(联网检索、配图等周边能力),给 GM 用。
> **边界**：自定义 MCP **不得碰规范态**(不能改 sheet/event/world/rule)——它的产出只作叙述/note 流回,不发呈现通知、副作用不进快照、走权限闸、归 out-of-canon 审计。这条边界是硬约束,不是建议。
> 更深的工程细节(hook 契约、审计实现)见 [开发指南/04-子系统设计/adapter与L3审计](../开发指南/04-子系统设计/adapter与L3审计.md)、[开发指南/04-子系统设计/玩家客户端 §8](../开发指南/04-子系统设计/玩家客户端.md)——本页只讲你需要做的操作。

## 什么时候需要它

你的团本想给 GM 加一个本身没有的能力——比如联网查资料、生成一张配图——又不想(也不该)让这个能力直接改动游戏的权威状态,就用自定义 MCP。

核心 dicelore MCP(骰子/人物卡/剧情/规则)是系统固定注入的规范态来源,锁定必需、不能关闭。自定义 MCP 是你额外加的周边能力,仅提供参考资料,归 out-of-canon(不参与 L3 审计、不发呈现通知、副作用不进存档快照)。

## 怎么接

在**配置页 → MCP 服务器**,"自定义 MCP"区右上角点「添加 MCP」,弹出的表单填:

- **实例名**:随便起一个好认的名字(如 `my-search`)
- **package**:MCP server 的 npm 包名(如 `@scope/mcp-server`)
- **command**:一般留默认 `npx` 即可
- **args**:传给命令的参数(如 `-y @scope/mcp-server`)
- **配置项**:该 MCP server 需要的环境变量(如 API key),按「加配置项」逐条填 key/value

填完点「添加」。添加成功后它会出现在自定义 MCP 列表里,每条右侧有个开关,可以随时启用/停用,不用重新填一遍。若列表条目显示"⚠ 未预拉",说明包还没在本地装好,首次使用可能要等一下。

## 记住这条边界

接完之后,你的自定义 MCP 能读到游戏正在发生什么(供它给出相关的内容),但**改不了** sheet(人物卡)、event(剧情记录)、world(设定卡池)、rule(规则)。它给 GM 的是"参考资料",不是"篡改权限"。
