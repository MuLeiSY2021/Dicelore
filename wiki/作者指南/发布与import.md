# 发布与 import

> **本页职责**：团本(Adventure)造好、校验通过之后,怎么让它成为一个别人能选来玩的东西。
> **格式细节**(团本包目录里每个文件/字段的完整规范)见 [开发指南/04-子系统设计/团本与manifest](../开发指南/04-子系统设计/团本与manifest.md),本页不重复,只讲操作流程。

## 团本包长什么样(一张地图)

团本是一个**目录**,里面全是纯文本(Markdown + CSV + 一份配置),没有二进制、可以整个当文件夹分发:

```
凡人修仙传/
├── manifest.yaml   # 这个团本的基本信息(名字/版本/用哪些玩法流程)
├── prologue.md     # 开场白——GM 开局第一句该做什么
├── world/          # 世界观、门派、NPC 人设
├── pools/          # 卡池、随机表
├── params/         # 判定分档之类的参数表
├── rules/          # 这个团本特有的规则
├── sheets/         # 开局时人物卡/世界的初始数值(可选)
└── fronts/         # 会自己推进的威胁/阵线(可选)
```

这些文件都是构建台帮你按格式写出来的,你不需要手写——上面只是给你一张地图,方便理解"团本"到底是什么。完整字段规范见 [团本与manifest](../开发指南/04-子系统设计/团本与manifest.md)。

## 提交并打上发布标签

在构建台的收尾阶段确认整包校验通过、内容都审阅满意之后,让 GM 帮你把这份内容**提交**成一个正式版本、**打上发布标签**。这一步之后,这个版本就定型了——之后再改团本会形成一个新版本,不影响已经打过标签的这一份。

## import:把团本灌进运行库

发布好的团本要"import"一次才能被拿来玩——这一步是把团本包里的内容读进运行时的游戏库里。import 的时候会**重新跑一次和构建期一样的校验**,所以只要你在构建台里校验通过,import 这一步不会因为格式问题失败。

import 完成后,这个团本就出现在可选团本列表里,玩家选中它就能开一局。

## 之后改团本、发新版本

- **规则可以随时更新**:如果只是改 `rules/` 里的判定规则,把版本号往上升一位就行,不影响已经在玩的存档。
- **改动结构(比如给卡池加一列、给人物卡加一个新属性)目前还没有自动迁移工具**——这类改动如果影响到已经在跑的存档,需要你自己判断兼容性,别轻易改已经发布过的结构。

如果还想给这个团本接一个额外的外部能力(比如联网查资料、配图),见 [DIY自定义MCP接入](DIY自定义MCP接入.md)。
