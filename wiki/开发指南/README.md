## 决策怎么记（无独立决策区）

决策不再单列一区，两处落地：

1. **「为什么这么设计」内嵌对应设计页的「决策与权衡」节**——决策与被否选项就近长在它影响的页里，改设计页时一并改决策（根治"只改决策不改设计页"的传播缝）。**这是唯一权威**，详情见对应设计页决策节。
2. **薄 [决策变更日志](决策变更日志.md)**——一行一条「日期 — 改了什么 — 链到哪页」，只留时间线、不含设计内容。

查某决策：看对应设计页「决策与权衡」节 → 时间线看决策变更日志（历史决策归档已撤）。

## 三条硬规矩

1. **单向推导**：下游页只引上游页（04 引 03、03 引 02、02 引 01，不反向）。要改上游才回头改下游。
2. **单源**：一件事只在一处是权威。词条只在 [术语表](../术语表.md)；问题账只在 [`docs/dev/plan/`](../../docs/dev/plan/)；决策只内嵌对应设计页。别复制，要么链接、要么沉淀。
3. **每页一职责**：每页开头一句"本页职责"。一句话说不清就拆页。
