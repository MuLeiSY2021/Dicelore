# 能力概览

> **本页职责**：不谈规划、只讲现在这一刻——玩家能做什么、作者能做什么。

## 玩家能做什么

现在的跑法是：用 `dicelore` CLI 建一局会话，再用 Claude Code 对话来玩——目前还没有面向普通玩家的图形化客户端，玩起来是命令行 + 对话的形式。

跑起来之后，框架会做到：

- **该掷骰的真的会掷骰、该给选项的真的会给选项**：判定成败或该由玩家做主的时刻，不会被 AI 一句话跳过或替玩家决定。
- **人物卡、剧情事件、世界设定跨回合持久化**：随时接上继续玩，不会因为对话变长就"失忆"。
- **能存档、读档**：每一轮自动记录检查点，可以读档回到之前的进度。

## 作者能做什么

通过"构建台"——一套换了工具的 Claude Code，跟里面的 lore GM 对话，同时在网页上审阅——把一份素材（小说、设定集）造成一个能被玩家选来玩的团本（Adventure）：

- **提炼世界观和 NPC**：从素材里整理出世界观、阵营背景，给重要角色写人设卡（需要具体数值的额外配数值卡）。
- **填卡池 / 随机池**：把素材里"抽到什么"一类的随机内容整理成卡池。
- **写机制规则**：这个团本特有的判定规则和对应的判定分档表。
- **收尾校验、发布**：整包校验通过后提交、打上发布标签，import 一次灌进运行库，团本就出现在玩家的可选列表里。
- **规则可随时更新**：改判定规则只需把版本号往上升一位，不影响已经在玩的存档；改动结构（比如给卡池加一列）目前还没有自动迁移工具，需要自己判断兼容性。
- **接自定义 MCP**：可以给团本额外接一个外部能力（比如联网查资料）。
