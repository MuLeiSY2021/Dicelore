# 从原著抽取团本内容的剧本

> 此文档在 `SKILL.md` 中以"参考文档"身份存在，仅当你需要策略细节时才读。

---

## 总体节奏

1. **摸源**：`ls materials/` + `wc -l -m` + `head` + `grep -c` 看清素材有什么、多大、什么结构。
2. **清洗分块**：剥噪声、按章节/主题切段，`Write` 到工作区（如 `clean/`）。
3. **分阶段导航**：每阶段用 `grep -n` 定位相关段落 → `Read` 读命中块 → 提炼成对应格式写入 Draft。

你的心智模型是**理解一个陌生代码库**：不整本读，而是先建结构地图，再按需跳读。命中后可读任意上下文、可反复收窄，直到攒够一段的素材再落笔。

---

## 摸源与清洗

先看清楚素材长什么样，再决定怎么切——没有万能脚本：

```bash
ls -la materials/                    # 文件清单 + 大小
wc -l -m materials/*                 # 行数 / 字数：判断能不能直接 Read，还是必须分块
head -80 materials/兽人冒险.md       # 头部：目录页？楼层号？作者按语？编码乱不乱？
tail -40 materials/兽人冒险.md       # 尾部：番外？完结感言？
grep -c '第.*章\|^楼主\|^#' materials/*  # 结构标记密度，判断按什么切
```

**常见噪声与剥法**（安价/论坛类素材尤其多）：

| 噪声 | 识别 | 剥法 |
|------|------|------|
| 投票楼 / 灌水短帖 | 行很短、重复颜文字、"顶""前排" | `awk 'length($0) >= 20'` 去短行 |
| 楼层号 / 时间戳 | `^#\d+`、`2015-` 之类前缀 | `sed 's/^#[0-9]* //'` / `grep -v` |
| 引用套引用 | `>>` `>` 引用前缀堆叠 | `grep -v '^>'` |

分块示例（供后续按块 `Read`，避免整本进 context）：

```bash
mkdir -p clean
# 去短行得正文
awk 'length($0) >= 20' materials/兽人冒险.md > clean/正文.md
# 按章节标记切成多个小文件
csplit -f clean/章- -b '%02d.md' clean/正文.md '/^第.*章/' '{*}'
ls clean/                            # 看切出多少块、各多大
```

清洗产物一律落 `clean/`（或 `notes/`）子目录，**别覆盖 `materials/` 原文**——原文是唯一可信来源，随时要回头核对。

## 按阶段的定位（Grep）策略

每阶段先在（清洗后的）素材里 `grep -n` 拿到命中行号，再 `Read` 读命中附近的段落。query 要**描述你想找的内容**，多给同义词/关键词以提高召回。

### 世界观

| 想找 | 推荐 grep 关键词 |
|------|-----------|
| 整体背景/时代/地理 | `grep -n '时代\|背景\|大陆\|国家\|地图'` |
| 门派/宗门信息 | `grep -n '门派\|宗门\|教派\|长老\|弟子\|规矩'` |
| 某具体地点 | `grep -n '黄枫谷'` |
| 修炼体系概览 | `grep -n '境界\|法力\|灵力\|灵根\|灵石'` |

抽到的世界观文档粒度建议：**一个地点/门派/阵营一篇**。层级太深则用子路径命名（`lore/门派/黄枫谷`）。

### NPC

| 想找 | 推荐 grep 关键词 |
|------|-----------|
| 人物出场+性格 | `grep -n '墨大夫'`（读命中块看性格/外貌/动机） |
| 关键配角关系 | `grep -n '韩立.*墨大夫\|墨大夫.*韩立'` |
| 反派/威胁人物 | `grep -n '反派\|敌人\|威胁\|大人物'` |

**NPC 识别信号**：有名字 + 反复出场（`grep -c '墨大夫'` 命中多）+ 对主角有影响 = 值得建文档。龙套不用专门建。

**是否需要 sheet 数值**：人设/性格 → `write_lore`；若开局就有明确战力/好感度数值 → `set_state(kind=npc)`。v1 建议少用 npc sheet，让运行时 AI 按需赋值。

### 卡池 / 随机表

| 想找 | 推荐 grep 关键词 |
|------|-----------|
| 物品/道具分级 | `grep -n '灵药\|灵器\|品级\|品质\|等级\|功效'` |
| 机缘/奇遇类型 | `grep -n '奇遇\|机缘\|秘境\|宝物\|邂逅'` |
| 灵根/天赋分布 | `grep -n '灵根\|天赋\|品级\|稀有\|概率'` |

**卡池设计原则**：weight 应反映原著中的稀有程度；若原著没给概率，按叙事稀缺程度估。

例子（灵根池）：
```
名称,品级,weight
天灵根,上品,1
异灵根,中品,8
三灵根,中品,15
四灵根,下品,25
五灵根,废灵根,51
```

### 机制规则

| 想找 | 推荐 grep 关键词 |
|------|-----------|
| 境界/阶梯 | `grep -n '境界\|练气\|筑基\|结丹\|突破'` |
| 突破条件 | `grep -n '突破\|天材\|丹药\|感悟\|感应'` |
| 战斗/斗法描写 | `grep -n '斗法\|战斗\|法术\|法宝\|灵力消耗'` |

规则文档写法：**先给运行时 AI 看的散文**，里面可以内嵌小表格（分档/曲线）。不需要专门建 params CSV，除非 flow skill 明确声明要吃某 params。

### 阵线 / Front

寻找适合做"后台威胁"的情节线：

| 信号 | 例子 |
|------|------|
| 世界级威胁，不跑也在推进 | 魔道入侵 / 宗门内乱 / 天地异变 |
| 有明显进度感（能量化） | "入侵已到第三阶段" / "已夺下三处要地" |
| 玩家的行动可以影响进度（正向或负向） | 驰援 = 减慢；放任 = 加速 |

每个 Front 对应一个威胁主线，凶兆阶梯建议 3~5 段，均匀分布在 min~max 区间。

## 常见陷阱

**陷阱 1：把整本原著贴进一个 write_lore**
原著几十万字不应该是一篇 lore 文档——它是 `materials/` 里的**素材库**（通过 Grep 定位 + Read 按需取用）。`write_lore` 写的是你提炼出的结构化内容，不是原文复制粘贴。

**陷阱 2：grep 关键词太窄**
`grep '韩立'` 会命中所有提到韩立的行，但你真正需要的可能是"韩立的灵根测试"这一场景。多给几个同义/近义关键词（`'韩立.*灵根\|资质测试'`），再看命中行号锁定要读的块。

**陷阱 3：没摸源就直接开写**
跳过阶段 0 直接 `write_lore` = 凭想象编原著。始终先 `ls`/`wc`/`head`/`grep` 摸清 `materials/` 结构，再动手——你引用的必须是原文。

**陷阱 4：NPC 数值 visible 写错**
NPC 暗数值（HP/战力/好感度）用 `visible:2`——AI 可见但玩家不可见。若写成 `visible:0` 则 AI 也看不见，无法利用这个数值做裁决。

**陷阱 5：清洗覆盖了 materials/ 原文**
清洗产物写到 `clean/`、`notes/` 之类子目录，别写回 `materials/`。原文是唯一可信来源，清洗判断错了还能回头重来。

## 完整流程例（凡人修仙传片段）

```bash
# 0. 摸源 + 清洗
ls -la materials/                                    # → materials/凡人修仙传.txt, 2.6MB
wc -l -m materials/凡人修仙传.txt                    # → 判断规模，必须分块
head -60 materials/凡人修仙传.txt                    # → 看结构：每章 "第N章 标题"
mkdir -p clean
csplit -f clean/章- -b '%03d.txt' materials/凡人修仙传.txt '/^第.*章/' '{*}'
```

```
# 1. manifest
set_manifest({ name:"凡人修仙传", id:"fanren-xiuxian" })

# 2. 世界观
grep -rn '七玄门\|黄枫谷\|大陆\|国家' clean/    # 定位命中块
→ Read 命中块
→ write_lore({ name:"lore/设定", content:"低武凡人流…" })

grep -rn '黄枫谷' clean/
→ Read 命中块
→ write_lore({ name:"lore/门派/黄枫谷", content:"…" })

# 3. NPC
grep -rn '墨大夫' clean/
→ Read 命中块（性格 / 外貌 / 炼药师身份 / 动机）
→ write_lore({ name:"lore/npc/墨大夫", content:"…" })

# 4. 卡池
grep -rn '灵根\|测试\|品级' clean/
→ Read 命中块
→ add_pool({ pool:"灵根", rows:[…] })

# 5. 机制
grep -rn '练气\|筑基\|结丹\|元婴\|突破' clean/
→ Read 命中块
→ write_rule({ name:"修炼体系", content:"…" })

# 6. Front
grep -rn '魔道\|入侵\|危机' clean/
→ Read 命中块
→ add_front({ id:"devil-invasion", … })

# 7. 开局状态
set_state({ cells:[{ entity:"韩立", kind:"player", attr:"资质", value:"五灵根", visible:1 }, …] })

# 8. 收口
validate({})
read({ section:"manifest" })
commit({ message:"凡人修仙传 v1.0 初建" })
tag({ commitId:"<returned>", label:"v1.0.0" })
```
