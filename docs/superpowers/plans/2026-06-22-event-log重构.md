# event → log 重构 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把运行时存储的 `event` 表/模块重命名重构为 `log`，新增 `is_moment` 列（默认 0），`event_fts` 虚表 → `log_fts`，`store/event.ts` → `store/log.ts`（`eventAppend/Recall/Since` → `logAppend/Recall/Since`，`EventRow/Input/Kind` → `LogRow/Input/Kind`）。expr/watcher 引用约定不变。全程 45 文件 209 测试 + tsc 0 绿。

**Architecture:** **就地两段改名**（非 state 那种 expand-contract 复制并存）。理由：`event` 是中央共享流水（`present/model`、`playerView`、`adapter/turnEnd`、`l3` 都读，几乎所有写路径都写），复制并存会让"写 log / 读 event"的中间态破绿。故：① **Task 1 = 表名改 `event`→`log`**（连同 `is_moment`、`event_fts`→`log_fts`、以及所有内嵌 SQL 引用），**函数/模块名暂留 `event*`**——所有调用经不变的 `eventAppend` 等命中改名后的 `log` 表，全程一致、绿。② **Task 2 = 标识符改名**（模块/函数/类型 + 全部 importers），纯改名、原子提交、绿。无"删除"步（是改名不是复制）。

**Tech Stack:** TypeScript（ESM，`.js` 导入后缀）、better-sqlite3、Vitest、`tsc --noEmit`。

## Global Constraints

- **expr/watcher 的 `{实体.attr}` 引用约定不改**：不得修改 `src/expr/*`。
- **MCP 工具名 `event_append`/`event_recall` 保留不变**（类比 state 重构保留 `sheet_*`）；`mcp/handlers/event.ts`、`mcp/schemas/event.ts` **文件名保留**，只改内部 import/函数调用。
- **`mcp/server.ts` 的 `CanonWriteEvent.kind` 联合里的 `"event"` 字面量保留**（它绑 `event_append` 工具的 canon-write 类别、非表名；改它会波及 `server.test.ts` 与 orchestrator 映射）。`server.test.ts` 第 28 行 `expect(...kind).toBe("event")` 不动。

- 每个任务结束：`npm test` + `npm run typecheck` 全绿（基线 45 文件 209 测试 + tsc 0），然后 commit。
- commit 用 scoped `git add`，只加本任务文件，**不要** `git add -A`。message 末尾：`Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`
