# 视图层投影落地 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把叙事层 spec §4 定义的 6 个命名视图（`player`/`npc`/`world`/`relation`/`clock`/`tension_board`）用 `CREATE VIEW` 投影落地、接进 `initSchema`，作为下游 toolgen 读工具的稳定逻辑列契约——这是头号债链路（第二批冻结令）依赖链的前置闸。

**Architecture:** 视图层零存储，按 `kind`+facet 投影 `state` 表 + UNION 聚合四张叙事表。新建 `store/views.ts` 导出 `initViews(db)`，在 `initSchema` 末尾调用（全仓 ~40 处 `initSchema` 调用零改动自动获得视图）。复用 `toolgen/view.ts` 的 `defineView`（带只读闸），但因 `defineView` 不幂等（`CREATE VIEW` 无 `IF NOT EXISTS`），`initViews` 内先 `DROP VIEW IF EXISTS` 再 `defineView` 以保证 `initSchema` 可重复调用。

**Tech Stack:** TypeScript + better-sqlite3 ^12.11.1 + vitest（co-located 测试，`packages/core/src/store/`）。

## Global Constraints

- 视图名必须匹配 `^\w+$`（`defineView` 闸）：`player`/`npc`/`world`/`relation`/`clock`/`tension_board` 均合法。
- 视图 SQL 体必须经 `assertReadOnlySelect`：首关键字 `SELECT`、不含分号、不含写关键字（`INSERT|UPDATE|DELETE|DROP|ALTER|CREATE|REPLACE|PRAGMA|ATTACH|DETACH|VACUUM`）。`CREATE VIEW` 由 `defineView` 自己 exec、不进 sql 参数。
- `StateKind = "player" | "npc" | "world"`（无 flag/clock kind——flag/clock 是 `state` 的行形态：`clock_min`/`clock_max` 非空 = clock 行）。
- `initSchema` 必须保持幂等（现有 `db.test.ts` 测了重复 `initSchema` 不报错）。
- spec §4 契约（行过滤 + 投影列）是稳定逻辑列契约，下游 toolgen 读编译只引用视图名+列，不碰物理表。
- 单源：不复制状态——视图是 `state`/叙事表的纯投影。
- **执行前置**：本 plan 在 worktree 内执行；worktree 内若 `packages/core/node_modules` 缺失，先 `npm install`（注意 `[worktree npm lock 坑]`：scoped `git add` 别 `-A`，提交前 `git checkout -- package-lock.json` 还原）。

## spec 张力点与决断（透明记录）

spec §4 `world` 视图备注「含 flag、未挂 front 的 clock」，但表格「行过滤」列只写 `kind=world`，且 `StateKind` 无 `flag` kind。**决断**：v1 严格按 spec §4 表格「行过滤」列实现——`world` 视图 = `kind='world'`；clock 行由独立的 `clock` 视图覆盖（`clock_min IS NOT NULL OR clock_max IS NOT NULL`，跨所有 kind）；front 引用的 clock 仍出现在 `clock` 视图，其 front 侧由 `tension_board` 列 active front 覆盖。理由：spec 表格是权威契约，备注是设计意图说明；`flag` 无独立 kind 无法在 SQL 层区分，留待 step③ 业务工具按 attr 约定处理。此决断在沉淀 wiki 时回注 spec §4。

```typescript
// Copyright (C) 2026 MuLeiSY2021
//
// This file is part of Dicelore.
//
// Dicelore is free software: you can redistribute it and/or modify it under
// the terms of the GNU Affero General Public License as published by the Free
// Software Foundation, either version 3 of the License, or (at your option)
// any later version. See <https://www.gnu.org/licenses/licenses/>.

import type { DB } from "./db.js";
import { defineView } from "../toolgen/view.js";

// 叙事层 spec §4 的命名视图（零存储，按 kind+facet 投影 state 表）。
// 视图名 + 投影列 = 对下游 toolgen 读编译公开的稳定逻辑列契约。
const VIEW_DEFS: ReadonlyArray<readonly [name: string, sql: string]> = [
  ["player", "SELECT entity, attr, value, visible FROM state WHERE kind='player'"],
  ["npc", "SELECT entity, attr, value, visible FROM state WHERE kind='npc'"],
  ["world", "SELECT entity, attr, value, visible FROM state WHERE kind='world'"],
  [
    "relation",
    "SELECT entity, rel_object, rel_dim, value, visible FROM state WHERE rel_object IS NOT NULL",
  ],
  [
    "clock",
    "SELECT entity, attr, value, clock_min, clock_max, clock_mode FROM state WHERE clock_min IS NOT NULL OR clock_max IS NOT NULL",
  ],
];

/** 投影全部命名视图。幂等：先 DROP VIEW IF EXISTS 再 defineView（defineView 本身不幂等）。 */
export function initViews(db: DB): void {
  for (const [name, sql] of VIEW_DEFS) {
    db.exec(`DROP VIEW IF EXISTS ${name}`);
    defineView(db, name, sql);
  }
}
```

Note: Task 1 暂不在此文件加 `tension_board`（Task 2 补）。`initViews` 已能跑 5 视图。
