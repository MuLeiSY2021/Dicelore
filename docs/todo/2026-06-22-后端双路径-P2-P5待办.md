### P3 · import 信任闸门 + 建库（依赖 P2）
- checkout 某团本版本 → **validator**（harvest 旧 2026-06-21 构建设计的校验规则 `checkRangeBands`/`checkCsvColumns` 等，重定向 DB，§9 rework）→ 物化出新 per-session 运行库；坏包被拦。
- **验收**：给 Catalog 一个版本能建出可跑运行库；坏声明拒绝。spec §3.3/§5/§7.3。

### P4 · git 单向投影（依赖 P2）
- isomorphic-git：`exportGit`（DB 版本日志 → 真 git history，tag→git tag）+ `importGit`（git → 新 DB 线性史，过 validator）。round-trip 测试。
- **承重**：DB 权威，git 仅进出口投影，**不双向镜像**。spec §6。
