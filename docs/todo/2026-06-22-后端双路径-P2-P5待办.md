## 待做（逐份成 plan，顺序：P2 → P3 → P5；P4 在 P2 后随时插）

### P2 · Catalog 团本包库 + 版本（根，ready）
- **落点**：`packages/core/src/store/` 旁新建 Catalog（与 `db.ts` 同栈、DB-only）。
- **内容**：DB schema（团本 + 版本表）；`resolveId`=UUIDv5(name)；线性 `commit`/`tag`/`history`/`checkout`/`list`（无分支）。
- **验收**：CRUD + 线性版本 + tag 单测。**依赖**：无。spec §3.2/§4/§7.3。

### P3 · import 信任闸门 + 建库（依赖 P2）
- checkout 某团本版本 → **validator**（harvest 旧 2026-06-21 构建设计的校验规则 `checkRangeBands`/`checkCsvColumns` 等，重定向 DB，§9 rework）→ 物化出新 per-session 运行库；坏包被拦。
- **验收**：给 Catalog 一个版本能建出可跑运行库；坏声明拒绝。spec §3.3/§5/§7.3。

### P4 · git 单向投影（依赖 P2）
- isomorphic-git：`exportGit`（DB 版本日志 → 真 git history，tag→git tag）+ `importGit`（git → 新 DB 线性史，过 validator）。round-trip 测试。
- **承重**：DB 权威，git 仅进出口投影，**不双向镜像**。spec §6。

### P5 · LoreSession 壳 + 上传 ingress（依赖 P1✓ + P2）
- `LoreSession`（壳，implements Session，kind=lore）+ `upload`/`extract`（ingress→抽取文本/blob 落 DB→弃原文件 + 注入框定占位）+ `api/lore.ts`。
- **不含** 构建 MCP 工具集 / 构建 skill 内容（见 DEFER）。spec §5/§7.4。
