## 1. PO 主实际在做的繁重工作（带实例）

**a) 维护人物卡 / 数值面板，且要持续更新。**

- 兽人团：开局让玩家逐项 roll 力量 / 敏捷 / 体质 / 感知 / 智力 / 魅力，自己换算战斗加值（"每一项 r 点均除以 2，为战斗时的加值"）；战斗后贴完整结算面板"你 HP 1/15 / 黑豹 HP 1/13（昏迷）/ 精灵 HP 4/16（昏迷）"，附规则脚注"单次伤害达 HP 一半以上会导致昏迷"；升级还要 roll 决定 HP 上升量、重算面板。
- 恶龙团：维护精确资源账本——"黄金 322000 / 食物 100 / 威慑力 1000 / 道具：夸尔羽符、守护卷轴、微视镜片、读心药水"；还要算"一头成年巨龙一天可吃掉自己体重一半的食物"这类供需平衡。

**b) 跟踪回合阶段 / 经营节奏。**

- 恶龙团有明确"阶段"概念：宣布"导入阶段可算弄完了，接下来是建设龙巢"，每行动轮给"最多三项"行动选项，周期性"决定下个阶段的重点目标"。
- 抽卡团把回合制写成正式规则："每回合行动顺序为：抽卡→融合（可选）→放置→事件结算"，"每十回合发生【影响世界之巨变】"，"每 15 回合出现一次世界意志卡牌"；每轮要打标题报阶段。

**c) 处理定时器 / 倒计时效果（埋点几回合后触发）。**

- 抽卡团大量卡牌带延迟触发："该卡牌存在时，将在三个回合后生成一张【圣骑士系列】卡牌"，PO 后来不得不在卡面显式写倒计时"剩余回合：3"。

**d) 收集与裁决投票 / roll，并处理平票。**

- 兽人团用"第一个 2,4,6"尾号规则、"fy"（附议）计票。平票时无机械规则，临场商量："你们咋刚好平票呢" → 玩家"直接 r 也行" → PO "那 r" → roll 出单数解决。

**e) 裁定行动合法性 / 即兴造规则。**

- 兽人团 PO 坦承规则拼凑："我只是用一部分五版和三版当参考方便 r 点"，后又"啊，好像是 coc 的规则，dnd 不常玩，忘了大半了"。遇规则未覆盖的行为当场发明判定："你用牛子去攻击她，你没有加值，精灵抵抗力类似 AC，大于等于则视为克服"。

**f) 记忆几十页前埋的伏笔 / 隐藏状态。**

- 抽卡团 PO **真的忘了**该触发的事件：玩家提醒"金獾有文明种子词条诶"，PO "该死的我都忘了金獾骑士团其实已经应该出现了"，随即补发"【金獾骑士团】诞生"。
- 恶龙团有玩家看不到的后台状态：苗床生育"分娩时后台进行体质检定"，俘虏有"固定意志值，每次检定，掉光就堕落为巨龙的忠实奴隶"——全靠 PO 心里记。

**g) 为每一次 roll 即兴生成内容。**

- 抽卡团最重：玩家 roll 出品质后，PO 要现编卡名 + 描述 + 特性，如"【起源斗士：生物 / 罕见 始动系列】…该卡牌不会被任何品质的卡牌摧毁，消失时为【始源大陆】任意卡牌提供升阶检定"。每张卡都是一段原创设定。

**h) 维护外部工具来对抗记忆负担。**

- 抽卡团 PO 被迫在主串外另开"茶水间"串 + 小程序记牌库；玩家直接提需求"po 有时间能发一下牌库吗，有些牌效果忘了，找起来也麻烦，可以放小程序上方便查找"。

## 2. "换成 AI 会很难做好"的地方

**a) 失败 / 坏结果：人类 PO 普遍硬着陆，让失败真的发生并变成剧情。**

- 兽人团开局战斗，玩家 roll 出 8，PO 算"8+3=11<12"，直接判**没打中**，把后果叙述成新困境："你没打着，黑豹闪过你的矛尖，回头咬住你左腿…精灵也醒了…一剑扎在你肩上"。主角最终带"HP 1/15"残血赢，失败被真实计入。
- 恶龙团有"大失败→NPC 当场死亡"的硬后果：玩家想救人，PO 判"没，大失败了，由于红龙极度愤怒，一尾巴给丫拍死了"。这正是 AI 倾向回避（软着陆、强行挽救）的地方。
- **关键设计自觉**：恶龙团 PO 事先声明失败的"烈度边界"——"巨龙拥有传奇抗性，豁免失败时可改为成功，一天三次。剧情方面大部分不会导致过于严重的失败"。即 PO **有意识把"该硬的地方硬、该兜底的地方兜底"分开**：战斗骰子照结果走（硬），剧情选择不让玩家一次脑抽就团灭（软）。**AI 难的不是"硬"或"软"本身，而是这种"事先声明规则、再一致执行"的分寸感。**

**b) 骰子不利时 PO 顶住、照骰子叙述。**

- 上面 8<12 的例子：玩家起哄"拉满"，PO 回"拉什么"，照样判失败。
- 平票也不偏袒，交给骰子。
- 抽卡团把不利后果写死在规则里——"若失败则选择一张素材破坏，若高于 90 大失败则素材全部破坏"，骰出坏结果就真毁卡，玩家自己 roll 完都认命"自裁"。

**c) 该投票 / 选择 vs 该掷骰，PO 怎么区分。**

- 规律清楚：**"主角自主决策"用选项 / 投票，"结果不确定 / 受运气或对抗影响"用骰子。** 兽人团给方向选项（"附近的森林 / 去草原抓 / 在路上蹲人"）让玩家选；"找到什么猎物"是随机表让玩家 roll；攻击命中也 roll。
- 两选项打平、属"无所谓谁对"时，PO 把投票降级成骰子（"直接 r 也行"）。
- 这种"何时让玩家做主、何时交给运气"的判断是隐性的、依赖叙事直觉。**AI 容易要么什么都 roll（剥夺玩家能动性）、要么什么都让选（消解风险感）。**

**d) 一致性与长期记忆。** 抽卡团 PO 自己"好多牌效果都忘了"、漏触发金獾，靠玩家和外部小程序兜底——这正是 AI 若无显式状态存储同样会犯的错，反过来说明**框架替 PO 维护权威状态**的价值。

## 3. 三个团各自的独特机制 & 对主持人的要求

**抽卡团：卡池 + 融合 + 互相引用的卡牌特性。**

- 机制：d100 定品质（1-15 平凡…91-100 无上）、融合检定（"r1d100，低于 50 成功…低于 10 大成功升一级…高于 90 大失败素材全毁"）、满 15 张触发文明生成、每 10 回合世界巨变。
- 对主持人的要求：(1) **海量原创**——每张卡现编名字 / 描述 / 特性；(2) **维护互相联动的"牌桌状态机"**——卡特性大量引用其它卡，放一张可能引发连锁结算，PO 要全盘追踪；(3) **当 live-service 设计师**——失衡了发"热补丁更新"重写规则。

**恶龙团：资源 + 派系经营。**

- 机制：黄金 / 食物 / 威慑力三资源，威慑力"决定村庄上供份额与可能到来的冒险者攻击强度"；定期征收贡品；地图上有血眼氏族、深渊掠夺者战团、巨胃部落、蛇神之牙等多个有兵种 / 靠山 / 地盘的派系。
- 对主持人的要求：(1) 维护**经济与人口收支平衡**（巨魔"一天吃掉近千公斤食物且极缺忠诚心"）；(2) **模拟一个有自主行为的派系世界**——玩家会做深度战略推演，PO 要给自洽的敌我实力评估并让世界做出反应。

**兽人团：D&D 式战斗 + 加值 / AC / 昏迷。**

- 机制：六维属性→战斗加值，攻击"r + 力量加值"对抗目标 AC / 抵抗力，半血昏迷，多实体战斗逐个结算，有 Action Surge 等招式。
- 对主持人的要求：实时做**多实体战斗的数值仲裁**；因是 homebrew，规则缺口要即兴补（CoC/3 版/5 版混搭、临场造判定）。

> **注**：三团都没出现机制化的"先攻顺序（initiative）"表——兽人团里"回合"只是口语化的"大战五十回合"。**机制化先攻在这三份语料里没找到。**

## 4. 玩家图什么 / 什么让他们好玩或弃坑

**好玩点 A：起哄、玩梗、把骰子结果当集体段子。**

- 兽人团四人 roll 出的选项"完美错开"，玩家乐"你们怎么做到的"；遇白皮精灵起哄"快进到被榨干""强大的敌人才能凸现兽人的力量"。骰子的随机性本身就是娱乐源。

**好玩点 B：深度策略推演与"集思广益"。**

- 恶龙团玩家自发写长篇战略："拉起一支军队…让红龙去跟大地精战帮谈…抓公主做人质"，另一位提供应链消耗战"切断粮草补给腐化农作物打击士气…坐收渔翁之利"。玩家当真正的策略游戏在玩，会主动引用 PO 给过的设定数据。PO 干脆开"茶水间"集思广益。

**好玩点 C：养成 / 数值成长的爽感。**

- 兽人团从八岁残血兽人打到有堡垒、有番号部落、给部落起名；恶龙团从光杆红龙发展到多兵种势力。长期成长是留存动力。

**弃坑 / 劝退信号：**

- **更新断续 + 等待**：抽卡团反复"今天没更新吗？""摸摸 po"，PO "今晚晚点，人还在外面"，最后玩家追问"咕了吗？"（疑似坑了）。**安科靠 PO 一人手动驱动，更新频率是生命线。**
- **冷场 / 人少**：恶龙团开局 PO 连说"人真少啊"；抽卡团核心仅 3 个 ID，PO 还发现"怎么物质卡基本没人抽呢"。人气不足直接影响投票 / roll 的参与度。
- **规则复杂到记不住、查不到**：抽卡团玩家明确抱怨"有些牌效果忘了，找起来麻烦"。**机制越深越好玩，但也越容易因"状态无人维护"而崩盘**——正是框架要解决的痛点。

## 给问题域的四点提示

1. **权威状态存储是刚需**：三团都在"手动记数值 / 记牌库 / 记倒计时"上吃力，抽卡团甚至漏触发、被迫自建小程序。框架替 PO 维护人物卡、资源账、卡池、定时器、隐藏值，价值最直接、最有出处。
2. **AI 当 PO 最难的不是算数，而是"声明后果在先、不利也照骰子执行"的硬着陆**，以及**有意识区分"该让玩家选 vs 该交给骰子"**——恶龙团 PO 的"传奇抗性兜底 + 剧情不强求失败 + 战斗照骰"分层设计是可借鉴范式。
3. **更新频率 / 在场感是留存命脉**：人类 PO 一咕就掉人，这反而是 AI-PO 的潜在优势（随时在线、即时结算）。
4. **规则即兴 / 热补丁能力**：homebrew 团靠 PO 临场造规则和发补丁维持平衡，框架需支持"运行中可调的规则层"，而非把规则写死。
