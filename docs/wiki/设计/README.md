## 决策怎么记（决策 B：无独立 ADR 区）

决策不再单列一区，三处落地：

1. **「为什么这么设计」内嵌对应设计页的「决策与权衡」节**——决策与被否选项就近长在它影响的页里，改设计页时一并改决策（根治"只改 ADR 不改设计页"的传播缝）。
2. **薄 [决策变更日志](决策变更日志.md)**——一行一条「日期 — 改了什么 — 链到哪页」，只留时间线、不含设计内容。
3. **[归档/历史决策日志](归档/历史决策日志.md)**——原 28 条 ADR README 原文逐字归档（非权威、以设计页为准），作历史与安全网。

查某决策：先看对应设计页「决策与权衡」节 → 时间线看决策变更日志 → 原文看归档。

## 三条硬规矩

1. **单向推导**：下游页只引上游页（04 引 03、03 引 02…，不反向）。要改上游才回头改下游。
2. **单源**：一件事只在一处是权威。词条只在 [术语表](../术语表.md)；问题账只在 [05](05-现状与计划/)；决策只内嵌对应设计页。别复制，要么链接、要么沉淀。
3. **每页一职责**：每页开头一句"本页职责"。一句话说不清就拆页。
