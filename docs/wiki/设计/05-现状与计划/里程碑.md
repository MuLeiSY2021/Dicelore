# 里程碑

> **本页人工维护**——AI 不得在**无人干预**下自行改动；在人明确指导、或调 `idea-to-roadmap` 抛点子（受人调用即干预）时，AI 可**追加/细化未来目标块（⬜）与「怎么实现」**，但**不擅自标 ✅ 完成、不改写已达成历史**（完成态由人在真达成那一刻定）。
> **里程碑 = 每个宏大目标 / 愿景**，拆成块、记「怎么实现」与进度。零碎进度与问题看 [backlog 三池](README.md)；为什么这么设计看 [ADR](../归档/历史决策日志.md)。

## 里程碑二 · 自建全栈玩家客户端（不依赖 Claude Code，用 Claude Agent SDK）

> 不再骑 Claude Code TUI，改用 **Claude Agent SDK** 程序化驱动 GM，**自己搭后端 + 前端界面**（完整独立玩家客户端，组件7）。**框架已搭起，多块细化设计 / 联调 / 验证未竟。**

- 🚧 **前后端框架搭建**：后端（orchestrator，Agent SDK headless）+ 前端（web）脚手架都搭起来了。
- ⬜ **前后端联调**：还没测试。
- 🚧 **前端页面细致设计**：只设计了一部分；**规则、session、团本页面都还没做细致设计**。
- ⬜ **配置页面**：没有细致设计，也没有前后端联调。
- ⬜ **GM 质量 eval 闭环（对照真实案例）**：把 `docs/research/` 抓来的真实安价案例（兽人冒险 / 抽卡 / 恶龙团）喂进**团本构建库**、和 lore GM 对话建出团本；**人工认可团本质量后**，让 **Claude 的 skill + MCP** 拿这个团本跑 eval、**落盘成 eval 报告**——对照**真实案例**看 GM 表现（**而非合成 baseline**）。*怎么实现*：[路线图第一批](路线图.md) + [backlog-core 主题F](backlog-core.md)。play-mcp 连真后端的 harness 已建。**注**：以目前认知**量化不可行**——闭环产出**定性报告**，量化口径待探索。
- ⬜ **可观测性 + 成本可观测性**：前/后端/core **三层日志分级统一**（结构化、不污染 stdio）+ **token 用量采集**（按 turn/session/agent 归因落库）+ **token/金钱消耗前端可视化**——这是后端可运营、也是计费/限流的数据前置。*怎么实现*：[路线图第三批](路线图.md) + [主题O / 成本可观测性](backlog-后端.md)。
- ⬜ **（后端完善后）让 Claude Code 反向操控后端**：构建一套与后端通信的 MCP + Skill，用它在真实团本 / 真实引擎中迭代 gm-core 的 skill 与 mcp——用真实跑团闭环打磨教条。

## 里程碑四 · 社区适配与持续迭代

> 发版后转入社区运营与长期演进。

- ⬜ **游戏生命周期完整性**：**终局判据 → 复盘模式**（GM 自调 game_end + 软判据 + 玩家主动结束权，终局后进复盘而非硬停，[ADR-0026](../归档/历史决策日志.md)）+ **快照/存档读档**（自动持久化，手动回滚/续命留 v2，[ADR-0017](../归档/历史决策日志.md)）。*怎么实现*：[路线图第四/五批](路线图.md)。快照 v1 已落 ✅，终局复盘已裁决待开发。
- ⬜ **后端性能与稳定性**：有待验证。含 **回合级事务 / 超时恢复**（GM 超时「脱困不恢复」修复：记 turn_start_seq、超时回滚回合 log），与快照接线合并。*怎么实现*：[路线图第三批](路线图.md) + [后端回合级事务主题](backlog-后端.md)。
- ⬜ **多租户**：概念还没有。租户模型 + SQLite 单文件隔离待裁（[路线图第五批](路线图.md) MT1）。
- ⬜ **安全**：完全没考虑。prompt injection 威胁建模 + key 后端托管 + 限流 + SSRF 白名单（[路线图第五批](路线图.md) SEC1/SEC2）。
- ⬜ **超开放的团本表现层**：让作者在框架下做出惊艳作品——**团本可注入富 UI（含 HTML）到 play 页面**（如崇祯模拟器式的大臣聊天/开会/战斗/国库栏），**UI 点选代替打字**。底层仍是文字冒险，只是表现层超开放（不改品类，[ADR-0022](../归档/历史决策日志.md)）。*怎么实现*：[backlog-前端 未来池](backlog-前端.md)（推翻原"限定 widget 集"中间路）。注：注入 HTML 有安全面（XSS），需净化/沙箱。其中"多角色对话"的实现方式**存疑待定**（见 [backlog-后端 未来池](backlog-后端.md)）。
- ⬜ **GM 能出图**：默认一个出图 subagent，GM 用自然语言驱动它生成图片，在对话框展示给玩家。*怎么实现*：[backlog-core / 后端 / 前端 未来池](backlog-core.md)。
- ⬜ **社区适配**：为社区接入做考虑、降低参与门槛。
- ⬜ **优化与迭代**：持续打磨、迭代项目。
- ⬜ **社区渠道**：社区群号 / 账号等运营渠道。
