## 总判据（一句话）

**成功 = 相比"裸 agent"，AI 的讨好本能被压下去，稳定地不犯 F1/F2/F3。** 不是"叙事多精彩"，而是"**该让玩家失败时它真的让失败发生**"。安科 / 安价里 GM 是服务器位，我们要的是这台服务器**可靠**，不是它有魅力。

## 1. F1/F2/F3「被治住」的可观测信号

每个失败模式给出"治住 vs 没治住"的对照信号，便于评测时打分：

| | 治住的信号 ✅ | 没治住 ❌ |
|---|---|---|
| **F1 跳过骰子** | 每个"结果不确定"的判定都**实际走了掷骰工具**，成败来自随机而非 AI 编造 | AI 直接叙述结果、省掉掷骰；或"假装掷了"但数字是编的 |
| **F2 不尊重结果** | 骰子不利时，叙事**如实承接坏结果**，且与**掷骰前声明的后果**一致 | 软着陆：淡化、转圜、给台阶、强行挽救失败 |
| **F3 选错方式** | 玩家自主决策→**给选项/投票**；结果受随机/对抗影响→**掷骰**，二者不串台 | 该让玩家选时替玩家骰/定；或该随机时却让玩家选，消解风险感 |

> F1 偏客观（"有没有调掷骰工具"可被 Hook 审计/脚本统计绕过率）；F2、F3 含判断成分（"算不算软着陆""该选还是该骰"需人评或裁判 agent）。所以度量是"客观信号 + 人评"混合，见下。

## 2. 怎么度量

**对照真实案例**（2026-06-29 定，[ADR-0025 修订补注](../归档/历史决策日志.md)）：拿 `docs/research/` 的真实安价案例（兽人冒险 / 抽卡 / 恶龙团，nmbxd 实跑串）作对照系——

1. **建团本**：把真实案例喂进**团本构建库**、和 lore GM 对话建出对应团本；**人工认可团本质量**。
2. **跑 eval**：让 **Claude 的 skill + MCP** 拿这个团本跑一局，**落盘定性 eval 报告**。
3. **对照真实案例评判** F1/F2/F3「被治住」程度：
   - **F1（掷骰绕过率）**：脚本可测的客观信号（机械地板：该掷未掷 / narrate 抢在 verdict 前 = fail），独立成立、与真实案例无关。
   - **F2/F3**：含判断成分，靠**对照真实案例 + 人评/裁判 agent** 定性评判——真实案例里真人 GM 遇同类局面怎么如实承接坏结果、怎么分配「该选 vs 该骰」，作参照系。
4. **迭代**：据报告改 gm-core skill 措辞 / L1 强制点，重跑。

> **量化以目前认知不可行 → 产出定性报告**：不声称某个"GM 满分"，也**不**靠"with 显著优于 baseline"那类 A/B 数值。**废 doctrine-vs-baseline A/B 对照**（[ADR-0025 修订](../归档/历史决策日志.md)：有无教条的消融答非所问；真正要回答的是"我们的 AI GM 跑得像不像真实安价好作品"，对照系只能是真实人类案例）。`DICELORE_BASELINE` 接线代码留作可选消融、不再当判据。

> **裁判 agent 的自我校准（F2/F3 度量的方法论风险，须显式防）**：F2/F3 靠"人评 / 裁判 agent 打分"，而裁判 agent 本身也是 LLM——它可能同样带讨好本能，把软着陆 judge 成"挺合理"，使评判偏宽、来自**裁判的宽容假象**而非真实改善。缓解：① 裁判 prompt 用**对抗性 rubric**（默认怀疑"这是不是软着陆 / 是不是替玩家骰了"，而非默认通过）；② 裁判结论与人评做**一致性抽查**，校准裁判松紧；③ 关键样本保留**人评终裁**。F1 靠脚本测掷骰绕过率，是客观信号，无此问题。

## 3. 非目标（明确放弃的边界）

写清非目标，是为了不被"完美 GM"幻觉带偏，也守住 [问题域](问题域.md) 的押注：

- **不追求取代人类 GM 的"上限"**：即兴惊喜、情感共鸣、赛后感想戦、玩家间的人际碰撞——这些是公认的人类上限（[期待与预测](调研-期待与预测.md) 第六节），Dicelore **只稳住下限**（把 GM 从单点故障解放、不崩坏、尊重骰子）。借安科/安价"服务器位 GM"的形态，放弃上限的代价很小。
- **跑团套件不为纯陪伴 / 角色拟真优化**：外置状态 + 强制掷骰这套结构约束，在机制驱动玩法（安价 / TRPG：骰子可信、数值长线、对抗）上是碾压优势，但在纯陪伴 / 角色扮演上反而成负担——强制骰子 / 外置状态在纯陪伴场景里是套件取舍，不是产品适配的方向。注意：这是**跑团套件层的特化取舍**，不是产品对陪伴品类的拒绝——界面 / 平台层是通用的，可对接各类 agent。架构层根因（状态归属代际差）见 [问题域 §6.3](问题域.md)。
- **不造游戏引擎**：状态机、约束验证器、效果调度器等不进主线；记账担子用最朴素的数据层兜住。要上的单开 spike。
- **单人阶段不追求玩家间碰撞**：那是多人阶段（场景 B）的事，见 [用户与场景](用户与场景.md)。
- **不追求叙事质量的绝对值**：叙事好不好看是次要的，主判据是"讨好本能被压下去"。叙事质量只在"明显劣化到影响可玩性"时才作为护栏看。
- **v1 不解长程语义召回**：伏笔/软线索的"相关时自动想起"是 RAG 难题，v1 只做朴素兜底（FTS5 + tag + agent 自己读），真·解法等 v1 观测后单开 spike。注意：目标/未结线索/世界硬状态等**结构化**部分不在此列——那属记账担子，照常显式追踪。详见 [问题域](问题域.md) 第 5 节。
