> **与其它页的边界（单源规矩，勿重复）**：
> - 各节 `TODO.md`（如 [03](../03-架构/TODO.md)/[04](../04-子系统设计/TODO.md)）= **节内** 做没做完的流水，本页不抄，只在里程碑里引「某节已定稿」。

## 怎么用这一节

1. **任何 session / 任何来源**冒出的问题，**只往这里的「问题总账」追加**（带固定字段），不再散落各处。
2. 隔一阵做一次 **去重 + 聚类 pass**：把同一架构病的多个症状归到一个**主题**下——*N 个 ticket 常常是 1 个决策*。
3. **反复出现 / 随规模恶化**的主题 = 最高架构优先级 → 开 ADR + 设计周期。
4. 拍了方案 → 写 ADR，问题条标 `→ ADR-00xx` 关闭。
