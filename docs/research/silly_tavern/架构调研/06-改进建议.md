## 〇 概览

### 0.1 改进总判

差距分析([04 §D.4](04-差距分析.md))与迁移方案([05 §0.1](05-迁移方案.md))已收敛到同一结论:dicelore 的**承重墙(L1 工具强制 / visible 横切 / 版本化 / watcher / 明暗骰)方向对、且比 silly_tavern 生态更工程化**;缺口集中在**机制下沉结构化载体、引擎强制门禁、裁决细化与场景注入**三块。改进不应碰架构骨架,而是在既有杠杆上**补结构化载体 + 把 L2 教条下沉为 L1 强制 + 收窄 rule 定位**。

### 0.2 改进三原则(承接 [05 §0.2](05-迁移方案.md))

1. **接口升级**:凡靠 prompt 教条 + regex 提取实现的,迁到工具调用 + 引擎内应用 + L1 强制(状态走 `sheet_update`,不走文本块)。
2. **层级升级**:凡靠 L2 prompt 教 LLM 自觉的,迁到 L1 引擎强制(watcher 自驱 / legality mask / 引擎参数化)。门禁不照搬 prompt。
3. **载体下沉**:凡把机制拍平进 rule/世界书散文的,迁到结构化可机械消费载体(pool / params / entity schema / front watcher)。

### 0.3 优先级骨架

| 级 | 头部项 | 一句话理由 |
|----|--------|-----------|
| **P0** | 机制下沉约束 + rule 定位收窄(+ entity schema + 一致性铁律) | 修 gacha-summon 那类真问题,性价比最高,不碰架构骨架,是 P1 门禁的数据地基 |
| **P1** | 引擎 L1 门禁层 + gm-core 教查 rule(+ L3 硬指标 + history compact) | 论文最强数据点 legal-masking −118 的落点;依赖 P0 就位后才能消费结构化边界 |
| **P2** | 团本级 skill 出口 + 难度预设 + 字面原则上升 Agenda | 依赖 P1 产出与 flows 断链修复;语义推理与品类特定编排,留 L2 references |

## 二 优先级排序与理由

### 2.1 P0:机制下沉约束 + rule 定位收窄(+ entity schema + 一致性铁律)

**为何这个序**:

1. **修真问题,性价比最高**。gacha-summon 实证([03 §五.1](03-现状分析.md))是当前最具体的活体病灶——构建 GM 把全部机制塞进单条 `write_rule` 散文,运行时 L1 强制退化为 L2 教条,anti-F1 防线被绕过。P0-1 直接修这个病根,且只动 build skill 教条 + validate warn + 补 params 工具面,不碰 play 侧架构骨架。

2. **是 P1 门禁的数据地基**。P1-1 引擎门禁要"据 entity schema(境界/能力边界)+ difficulty 参数计算合法行动集"([05 §1.4](05-迁移方案.md)),没有 P0-1 机制下沉(品质表/分档表落 pool/params)和 P0-3 entity schema(字段边界),门禁无数据可判合法性——门禁建在沙地上。故 P0 是 P1 的前置依赖。

3. **不碰架构骨架**。P0 四项都是"补结构化载体 + 补 L1 校验 + 收窄定位",不改变四域物理 schema、不改变 L1/L2/L3 三杠杆结构、不改变 play/build 双侧隔离。是最低风险的增量。

4. **多源共指优先级最高**。[04 §D.1](04-差距分析.md) 三项多源共指(裁决成体系含代价相称 / 字面原则与一致性铁律 / 机制下沉结构化载体)中,P0 覆盖共指#3(机制下沉)与共指#2 的 L1 部分(一致性铁律);人物卡工程实践与论文数据正当性汇合点,优先级最高。

### 2.2 P1:引擎 L1 门禁层 + gm-core 教查 rule(+ L3 硬指标 + history compact)

**为何这个序**:

1. **论文最强数据点的落点**。legal-action masking −118(2.1× 相邻模型差)是 [02 §二.5](02-论文结论.md) 最强数据点,"把合法性从模型推理移到引擎强制才是 2.1× 杠杆,远超状态格式 −33"。dicelore 的 L3 审计薄([03 §三.2](03-现状分析.md))正好是这个杠杆的缺口([04 §D.3](04-差距分析.md) 单源论文指向)。P1-1 是这个数据点的直接落点。

2. **依赖 P0 就位后才能消费结构化边界**。门禁要判"物理可行性 + 难度合规",需 entity schema(境界/能力边界,P0-3)+ 机制下沉(品质表/分档表落 pool/params,P0-1)提供数据。P0 未就位则门禁无据可判。故 P1 排在 P0 之后。

3. **gm-core 教查 rule 是 P0 机制下沉的运行时配合**。机制下沉后,GM 须知道"机制在哪→先查 rule"才能拿到结构化载体引用;P1-2 补这个 L2 教条,与 P0-1 配合才闭环。单独做 P1-2(不先 P0-1)则教查 rule 仍只能拿到散文,无意义。

4. **L3 硬指标 + history compact 是论文数据支撑的次级杠杆**。L3 审计薄(共指 [04 §D.3](04-差距分析.md))与 history −115([02 §二.3](02-论文结论.md),仅次于 legal-masking)都是论文强支撑,但前者依赖 P1-1 门禁产出可审计标记,后者 v1 单人短局可酌情后置,故同列 P1 但内序靠后。

5. **不碰骨架,但杠杆大**。P1 仍在既有 L1/L3 杠杆上加内容(新 legality check 工具 + auditTurn 扩充 + history 工具实现),不改变三杠杆结构。

### 2.3 P2:团本级 skill 出口 + 难度预设 + 字面原则上升 Agenda

**为何这个序**:

1. **依赖 P1 产出与 flows 断链修复**。难度预设(P2-2)4 模块中天道衍算/越阶天堑是 P1-1 门禁参数,众生心智是 P0-3 entity schema——P1 未就位则难度无处落;团本级 skill 出口(P2-1)依赖 flows 断链四段全修([03 §四.4](03-现状分析.md)),且 COT 语义推理部分不能落 L1(会变 FG-states 过度调用,[02 §一.3](02-论文结论.md)),只能留 L2 references。故 P2 排在 P1 之后。

2. **语义推理与品类特定编排,留 L2 references**。[05 §3.1](05-迁移方案.md) 张力判据:凡可机械判定的→L1(P0/P1 已覆盖);凡需语义推理的(形势紧迫度/叙事走向/事件选择)→L2 flow skill references(P2-1)。P2 是 L2 层的补充,不是 L1 强制,杠杆低于 P0/P1。

3. **字面原则是 L2 软约束,收益低于 P0 结构化校验**。[04 §D.1 共指#2](04-差距分析.md) 中,字面原则(防脑补)的论文支撑是 PAYADOR 结构化校验([02 §四.1](02-论文结论.md))——即 P0-4 一致性铁律的 L1 校验;字面原则措辞本身是 L2 软约束,GM 读了未必照做([03 §三.2](03-现状分析.md) F4 实证"读了教条仍 0 次明骰")。故字面原则上升 Agenda 列 P2,不与 P0 结构化校验争优先级。

4. **风险/收益比允许后置**。P2 三项都是 L2 层(措辞/references/富字段),不影响 L1 强制承重墙;v1 发版前可酌情排,不阻塞 anti-F1/F2 防线。

## 四 不建议做

### 4.1 万物皆文本(论文 FG-gen 0.267 直接否定)

- **不做**:状态以文本生成、不走 schema-validated function call;照搬万域 `<UpdateVariable>` 文本块 + regex 提取 + 外部 MVU 引擎应用。
- **依据**:[04 §A.2](04-差距分析.md) 否定 / [02 §一.2](02-论文结论.md) FG-gen 单测 0.267 最低、一致性 3.698 < FG-all 4.388(p=0.0025)/ [05 §2.3](05-迁移方案.md) 不做项。
- **dicelore 已规避**:走 `sheet_update` 工具调用直接写结构化 state 表(FG-all 路线,[04 §C.2](04-差距分析.md) 符合),不退回文本生成。

### 4.2 照搬 COT 7 步 prompt 堆叠(违背 L1 工具强制原则)

- **不做**:把万域 6987 字 COT 协议整体照搬成 prompt 堆叠,靠 io=9002 注入优先级确保"协议不可跳过"。
- **依据**:[04 §A.2](04-差距分析.md) 否定其层级(门禁靠 prompt 教 LLM 自觉过 = L2 教条)/ [02 §二.5](02-论文结论.md) PTCG −118 指向 L1 引擎强制,远超 L2 / [05 §2.3](05-迁移方案.md) 不做项。
- **判据**:COT 按"可机械判定 vs 需语义推理"拆解([05 §1.2](05-迁移方案.md))——机械部分(合法性/代价/字段边界/同步性)落 L1 工具,语义部分(形势/叙事走向)落 L2 references;不可整体照搬成 prompt。

### 4.3 让 rule 承载程序性决策协议(rule 是陈述性,非程序性)

- **不做**:把"骰值→档位→后果"映射、融合结算流程、周期事件触发逻辑等**程序性决策协议**塞进 rule 散文,让 GM 整段读 + 自行套用 + 心算映射。
