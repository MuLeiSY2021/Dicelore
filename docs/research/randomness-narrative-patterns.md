## 一 随机密度量化：`r` / `fy` / 纯叙事 的比例

| 文件 | 掷骰事件（r/rNdM/NN[lo,hi]） | 投票 `fy` | GM 选项菜单(`＞`行) | 检定/判定/DC 提及 |
|------|------|------|------|------|
| 兽人（dnd） | **≈372**（裸 r 为主） | 15 | 1054 | 6 |
| 抽卡 | **≈162**（6 裸 + 156 渲染值） | 22 | 1031 | 114 |
| 恶龙团（苗床安价） | **≈148**（69 裸 + 79 渲染值） | **98** | 1420 | 12 |

### 结论（校正用户的"一半以上靠随机"体感）

1. **随机密度强烈依赖 genre，不是单一数字**：
   - **dnd 风（兽人）**：掷骰是主引擎，`fy` 几乎不用（15 次）——每次玩家行动几乎都伴一个 `r`，剧情靠"行动→骰→后果"链推。
   - **抽卡团**：随机是世界生成的地基（每回合抽卡 `NN[1,100]` 定品质 + 大量融合/升阶/文明检定，114 处"检定"），但骰值多由 GM 侧解析（玩家抽、GM 结算）。
   - **苗床安价（恶龙团）**：**投票（98 fy）压过掷骰**——方向靠集体 `fy` 定，掷骰只在结果不确定的对抗/检定处出现（148 次仍不少）。

2. **对"偏给选项"观察的精确修正**：真人串里 `＞选项菜单` 行数确实巨大（1000+），**但这不等于 dicegm 的 `resolve_choice`**——见 §二/§四：真人的"选项"是**开放式提案征集**（`＞其他` 常驻 + 玩家自撰提案被 `fy`），且**只用于"玩家自主决定往哪走"的岔路**；一旦进入"结果不确定且失败有意义"的拍子（战斗、检定、抽卡、随机遭遇），真人 GM **立刻转随机**，绝不用选项菜单替代掷骰。**dicegm 的病不是"用了 resolve_choice"，而是把 resolve_choice 用到了本该掷骰的拍子上**（闸 B 该骰处退回给选项）。

3. **可用作 skill 的密度参照**：在任一 genre 里，"行动/推进拍"落到掷骰或抽卡的比例都远高于 dicegm 当前实跑。dnd 风近乎"每个玩家主动行动一掷"；抽卡"每回合必抽 + 每次融合/升阶必检定"；安价即使投票主导，对抗拍仍必掷。

## 二 随机触发时机：真人 GM 何时让 `r`（明）/ 自己 `r`（暗）/ `fy`（投票）

真人实践与 SKILL 的 **闸 A/闸 B/谁掷表高度吻合**——taxonomy 是对的，缺的是"密度默认值"（见 §六）。逐类举证：

### 2.1 明骰（玩家 `r` 自己的命）——玩家主动行动 / 建卡

- **玩家主动攻击**：兽人 No.38589458 GM 叙述玩家偷袭黑豹后写下 `r，力量+3`，玩家们各自打 `r`（No.38589483/38589515/38589523），GM 取值结算 No.38589888「8+3=11<12 你没打着」。→ 完全对应 SKILL「玩家主动行动的检定 → 明骰 `resolve_contest_open`」。
- **建卡六维**：兽人主串 No.38582339「首先，为我们的主角 r 个身体素质／每一项 r 点均除以 2，为战斗时的加值／力量 敏捷 体质 感知 智力 魅力」，No.38582369「本项为一尾，本团平时默认 2 尾和」——玩家逐项打 `r` 掷自己的属性。→ 对应 SKILL「玩家掷自己的属性/建卡 → 明骰让玩家自己掷…禁止 `sheet_update` 硬编数值属性」。真人**从不由 GM 替玩家定属性**，铁律级一致。
- **玩家触发的随机遭遇表**：兽人 No.38592398 GM 在旅途拍子写「到家之前 r／1~6 无事／7~9 遇到了狼／0 同伴嫉妒你决定干你」，玩家打 `r`（No.38592414）。→ 结果形状是 label 档位，玩家触发 = 明骰 `resolve_outcome_open`。

### 2.2 暗骰（GM 自己 `r`）——NPC / 世界属性 / 隐藏检定

- **世界/NPC 属性生成**：兽人 No.38592076「这里 r 是暗骰啦( ﾟ∀。)」；No.38592426「由于喑骰了一下像人程度为 0（取 10），所以这女地精长这样」——女地精"像人程度"是世界属性，GM 暗骰决定其外观、只把结果叙述出来、不让玩家触发。→ 对应 SKILL「NPC/世界/暗检定 → 暗骰 `resolve_*_hidden`／`world_sample`，引擎自动掷」。
- 明/暗的分野在真人串里是**自觉且明示**的（GM 会特意标"这里 r 是暗骰"），佐证 SKILL 的明暗二分不是过度设计。

### 2.3 投票 `fy`——集体定方向（安价）

- 恶龙团主串 No.54995176「为主角起个名先（十分钟内 fy 最多或尾号最大的）」——预先声明"fy 最多 / 尾号最大"为裁决规则（tiebreak 用尾号=随机）。
- 恶龙团 No.55120451/55120479 玩家自撰提案「女王和一部分士兵跟咱们走，留点蜘蛛耕田养蛛」→ No.55121789 他人「fy，带走高端战力，剩下的留下种田」→ GM No.55128517 据票数叙述该方向。→ 对应 SKILL「安价/投票定方向 → `dicelore-flow-anka`」，但 dicegm 的 `resolve_choice` **只提供固定菜单**，缺"玩家自撰提案 + 他人附议"这层（见 §四/§六 缺口）。

**偏差小结**：SKILL 的闸 A/闸 B/谁掷表 **在真人实践里全部命中**；偏差不在分类，在于 dicegm 实跑时**闸 B 该骰处的判定过于保守**，退回 `resolve_choice`。

## 三 骰后叙述手法：坏结果怎么咬下去 / 好结果怎么不给满

### 3.1 fail-forward（坏结果咬下去、不软着陆、且推进）

- **兽人 No.38589888**（攻击 miss）：「8+3=11<12 你没打着，黑豹一下子闪过你的矛尖，回头咬住你的左腿…精灵也醒过来了，她慌张的抽出短剑刺向你…扎在了你肩上」——miss 不是"什么都没发生"：黑豹反咬、睡着的精灵被惊醒加入战斗，**局面恶化并打开新战线**，随后紧接 `＞失手了！跑路／＞淦你娘看招／＞其他`。这是 F2 下边界（fail-forward）的教科书样本：坏结果 = 新威胁 + 新岔路，不停摆。
- **抽卡 No.67957927**（融合检定失败）：「融合失败，选择一个素材摧毁」——按主串 No.67916539 预声明的规则「融合检定 r1d100，低于 50 成功，失败则选择一张素材破坏，>90 大失败素材全破坏」硬执行，**后果先声明、骰后照办、不回护**。对应 F2 上边界（anti-讨好）+ `references/consequences.md` 的"后果先声明再一致执行"。

### 3.2 anti-讨好（好结果不给满 / 上限焊死）

- **兽人 No.38589528→38589593**：玩家喊「拉满(σﾟдﾟ)σ」想要满值，GM 回「拉什么( ﾟ∀。)」，玩家自嘲「草，最高不是 10 吗」——**GM 拒绝为讨好玩家突破骰值上限**。机械层的 anti-讨好：点数恒由引擎/骰算，GM 不加码。对应 Agenda 0「诚实仲裁者，不是取悦者」+ F1「你给不出引擎内的真值」。

### 3.3 三档结果 + 大成功/大失败

- 抽卡主串 No.67916539 的融合规则天然是**四档**（<10 大成功升级 / <50 成功 / 失败破坏素材 / >90 大失败全破坏）——对应 `references/consequences.md` 三档结果 + `resolve_outcome` bands。真人对"大成功/大失败"的额外档处理，可作 bands 设计的正典佐证。

## 四 随机驱动的剧情结构：一连串随机堆出转折（play to find out）

真人安价**不预设剧本**，用"随机→新局面→再随机"链推进，GM 只做仲裁 + 叙述：

- **兽人**：建卡随机（r 六维）→ 遭遇随机（黑豹/精灵，No.38589458）→ 战斗随机（每次 attack `r`）→ 结果随机（miss 反咬 No.38589888、伤害 `8+3` `12+12` No.38590332）→ 新局面（俘虏精灵、遇地精、旅途随机遭遇表 No.38592398）→ … GM 从不知道下一拍走向，**"play to find out"是被随机强制的**。这正是 dicegm Agenda 第 3 条想要、但实跑达不到的密度。
- **抽卡**：每回合"抽卡 `NN[1,100]`→定品质→GM 据品质+类型+系列现场生成卡牌内容"（No.67916667 `33[1,100]` 生物→罕见→No.67916694 生成「火焰魔导师」），卡牌间又靠检定互相触发（升阶/融合/文明生成检定，主串 No.67916539/126）——**世界形态完全由随机链累积生成**，GM 是随机结果的叙述者+规则执行者。
- **恶龙团**：方向靠 `fy`（No.55120479 迁徙决策），对抗靠掷骰，两者交替堆出"红龙征服群山"的剧情——**投票定"往哪推"，随机定"推的结果"**，二者分工清晰。

**结构性发现**：真人 GM 的"选项菜单"（`＞…＞其他`）是**推进拍的方向征集**（play-to-find-out 的入口），随机是**结果裁决**；两者不是竞争关系而是分层。dicegm 把二者混淆——在该"裁决结果"的拍子上给了"征集方向"的 `resolve_choice`。

## 五 建卡 / 抽卡随机链路：开局随机如何锚定后续

- **建卡（兽人）**：开局 r 六维 → 每项 /2 = 战斗加值（主串 No.38582339）→ 该加值贯穿全部后续战斗结算（No.38589888 的 `+3` 力量加值、No.38590332 伤害）。**开局随机一次性锚定人物战力基线，后续所有掷骰在此基线上展开**。对接 `dicelore-flow-anka` 的建卡轮 + SKILL「开局/建卡轮先 r 六维明骰建卡…再尽快推进到第一个需要裁决的场面」。真人做法印证"别纯建卡铺场景空转"：兽人串建完卡数帖内即进狩猎遭遇。
- **抽卡（抽卡团）**：开局抽卡 `NN[1,100]` → 品质表（主串 No.67916539：1-15 平凡…91-100 无上）→ 卡牌进入场域后靠特质 + 检定互相作用生成文明（No.67916539 文明生成检定）。**开局抽卡不只是发牌，是世界种子**：抽到的卡决定了后续可解锁的系列（No.67916652「系列解锁：魔法系列」）与文明走向。对接 `dicelore-flow-gacha`。
- **共性**：开局随机 = 后续剧情的**锚点/约束集**，不是孤立的开场仪式。SKILL 的"建卡轮尽快推进到裁决场面"可补一句"开局随机结果须作为后续裁决的锚定输入被引用"。

## 七 skill 改进建议（RD-2 交付 · 给 RD-3 的施工草案）

> 以下为**建议 + diff 草案**，不在本节点落地（RD-3 改 SKILL.md，RD-5 收口措辞，RD-4 eval 验证）。所有措辞标 `eval-pending`，终稿靠 skill-creator eval-loop。

### 7.1 Moves 改动点：强化"先问该不该骰、别默认给选项"

**问题根因**（§一.2 + G3）：dicegm 推进叙事时默认走 `resolve_choice`，把"结果裁决拍"误当"方向征集拍"。

**建议**：在 SKILL.md「Moves」段首、两个极端防护之后，补一条**反默认倾向**的显式提示，并区分"方向征集" vs "结果裁决"：

```diff
 两个极端都要防:**别什么都骰**(该让玩家做主时替玩家骰→剥夺能动性)/ **别什么都让选**(该交给运气/对抗时让玩家选→消解风险)。
+
+> **反保守默认**(真人语料校准):推进叙事时先问「这一拍该不该骰/抽」,**再**问「要不要给选项」——次序不能反。真人跑团里(见 `references/randomness-narrative.md`)dnd 风近乎每个玩家主动行动一掷、抽卡每回合必抽、即便安价投票主导的团对抗拍仍必掷;`resolve_choice` 只用于「玩家自主决定往哪走」的**方向岔路**,不是每个推进拍的形式。**闸 B 判定"该骰"时直接骰,别退回给选项菜单**——那是本 skill 最常见的失效模式。
```

**并在闸 A 补一句**区分方向征集 vs 结果裁决（对应 G3/§四）：

```diff
 - 不是玩家自主(命中没、抽到啥、掉多少血)→ 进闸 B。
+ - **辨向**:`resolve_choice` 是「往哪走」的方向征集(play-to-find-out 入口);「走了之后成不成/得多少」是结果裁决,归闸 B 掷骰。同一拍常两者都有——先征集方向、再对不确定结果掷骰,别用一个 `resolve_choice` 把该骰的结果也一起"选"掉。
```

### 7.2 Principles 改动点：F1 补真人榜样锚

```diff
-- **F1 必掷骰**:该裁决处必经裁决工具——随机/取真值在引擎内、你给不出真值。识别该裁决的时机,别用散文绕过。*why*:玩家的风险感来自结果不可由你编造。
+- **F1 必掷骰**:该裁决处必经裁决工具——随机/取真值在引擎内、你给不出真值。识别该裁决的时机,别用散文绕过、**更别用 `resolve_choice` 绕过**。*why*:玩家的风险感来自结果不可由你编造。真人密度榜样(dnd 每行动一掷 / 抽卡每回合必抽 / 对抗必掷)+ 骰后叙述范本 → `references/randomness-narrative.md`。
```

**"明骰默认"条**已到位，无需改；可在 F2 末尾把 consequences 指引扩到新 reference：

```diff
-*why*:失败被真实计入才有重量;但失败让故事停摆同样糟。手法 → `references/consequences.md`。
+*why*:失败被真实计入才有重量;但失败让故事停摆同样糟。手法 → `references/consequences.md`;真人骰后叙述 worked examples(miss 反咬开新战线 / 融合失败毁素材)→ `references/randomness-narrative.md`。
```

### 7.3 新增 `references/randomness-narrative.md`（RD-3 建）

裁决 §二/C5 拍定。建议纲要（内容内联本文精华 + 引用帖段）：

1. **密度参照表**（§一）：三 genre 的骰/投票/菜单量级，给 AI 一个"该骰长什么样"的锚。
2. **明/暗/投票 触发速查**（§二）：附真人帖段（明骰 No.38589458；暗骰 No.38592426；投票 No.54995176/55120479）。
3. **骰后叙述 worked examples**（§三）：
