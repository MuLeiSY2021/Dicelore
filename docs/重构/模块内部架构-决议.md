# 各模块内部架构优化——决议（基于 3 份并发调研）

> 重排+storage-port 全绿后,对 frontend/backend/harness **内部**结构按成熟惯例优化的决议。
> 共享不变量(三模块通用):**包的公共 barrel/exports 保持不变 → 外部包零改动、影响锁在包内**;纯机械「移文件+改内部 import+更新 barrel 相对路径」,typecheck+test 兜底。落地后沉 wiki(03 架构 / 04 子系统),本文随之退役。

---

## frontend（@dicelore/frontend，React+Vite）
调研依据:bulletproof-react / Wieruch / developerway / React 官方。结论:**轻量 feature 切片 + shared 公共层 + app 入口层**(不照搬 bulletproof 全套空目录,YAGNI)。

**采纳**：
- `@/*` 路径别名(vite resolve.alias `@`→/src + tsconfig paths);消除 `../../` 脆弱相对路径。
- `api/client.ts`(201 行混 5 域)拆:`shared/api/http.ts`(fetch 基建+health) + 各 `features/<域>/api.ts`。
- **feature 切片**:`src/{app(main/App/router),shared(api/i18n/theme/settings/icons),features/{play,catalog,build,home,config},shell,styles}`。play 域聚拢(PlayPage+useSession+Markdown+api)。
**守**:**不引入 feature `index.ts` 桶文件**(害 Vite tree-shaking);测试 colocation 保持。
**跳过**(本轮不做):CSS 拆分(牵墨金视觉,另议)、i18n 词典分片、`shared/ui`(YAGNI 缓建)。

## backend（@dicelore/backend，Node 存储+服务）
调研依据:domain-driven-hexagon / Fowler dependency-composition / MS DDD / 「按域非按层」。结论:**store/ 按业务域内分组;其余已对齐惯例不动**。

**采纳**：
- `store/` 域内分组:`sheet/`(state,mutate,visibility)、`event/`(record,history)、`world/`(world,rule,anchor)、`narrative/`(front,plotline,foreshadow,watcher)、`interaction/`(choice,pendingRoll);`db/views/fts*` + `snapshot/usage/evalCtx` 留 store 顶层(基础设施/横切)。
**不动**(已领先惯例):storage-port(`sessionBackend.ts` + `@dicelore/interface`,教科书 ports-adapters)、`api` transport 薄层、`expr/resolve/present` 引擎切分。
**降级文档**:`catalog/`、`build/` 已是事实独立子域,wiki 标注即可、不搬。
**跳过**:`composition/` 目录、子 barrel 拆分、`infra/` 归类(低价值 churn)。

## harness（@dicelore/harness，agent 运行时）
调研依据:Claude Agent SDK(sessions+hooks+role 分离) / LangGraph(per-agent 自包含) / hexagonal。结论(读 import 图实证):**mcp/ 与 adapter/ 实为 dicegm 私有(loregm 零引用),错抬成顶层;runtime/ 才是真共享骨架**。

**采纳**(让目录表达真实依赖、两条 GM 线对称自包含)：
- `src/mcp/` → `src/dicegm/mcp/`(in-play 工具面是 dice 私有)。
- `src/adapter/` → `src/dicegm/adapter/`(GM signpost/L3/团本 skill 安装,全 dice 私有)。
- skills 按角色归位:`harness/skills/{gm-core,flow-*}` → `src/dicegm/skills/`;`dicelore-build-pack` → `src/loregm/skills/`。归属从「文件名前缀约定」升级为目录结构(`skills-structure.test` 可钉)。
- `runtime/` 不动(Session/registry/streamTurn/agent/wsHub/reasoning,真共享)。
**运行期注意点**(唯一非纯机械):skills 路径解析——`openingPrompt` gmCoreDir/buildPackDir 改「同角色线根 `./skills/...`」;`adapter/init.ts` SKILLS_SRC、cwd 兜底、`skills-structure.test`、`mcp/main.ts`(→dicegm/mcp/main.ts)的 `dicelore:mcp` script + `templates.ts` mcpJson 路径,一并改并验。
**Follow-up（#7/打包期）**:skills 是运行期 `.md` 资产、当前无 build 步(main 指 src),放 `src/` 下 OK;**将来若加 bun --compile/SEA 打包,须确保 skills `.md` 进产物**。
**非本任务**:别为 loregm 对称而造它不需要的件(lore 是 REST-only、无 WS/gate),那是功能差异非结构债。
