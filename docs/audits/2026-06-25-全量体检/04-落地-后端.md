# 后端 · 落地
> 上游文档：02-设计-架构.md
> 本文档：作为后端工程师，对 apps/orchestrator + packages/core 的实现落地负责——评估 GM 调用链健壮性、并发竞态、错误处理与降级、快照/回滚正确性、LLM 调用可观测性、后端测试缺口，并承接架构层 port 契约与缝 A/缝 B 约定，验证实现是否落地了架构契约、给前端暴露的契约是否与前端假设一致。

## 上游回放

接住 02-设计-架构.md 的关键锚定：

- **ARCH-001/008（零鉴权 P0）**：架构层定了缝 B「需会话路由+鉴权」+ ADR-0018 ③ Web 壳「需会话路由+鉴权」，但实现 Hono app 零中间件、sessionId 即权限、lore 路径 Catalog commit/tag 零鉴权。架构定了、实现零承接。
- **ARCH-002（快照未接线 P1）**：ADR-0017 快照/回滚设计自洽，但 `db.ts initSchema` 无 snapshot 表（注释明示"并行未 rebase"）、core 无 checkpoint/restore 原语、`DiceSession.turnEnd` 不调 checkpoint()。
- **ARCH-003（toolgen 零接线 P0）**：present 层有 tensionBoard 聚合函数，但 `TOOLS` 运行时清单不含 front/plotline/foreshadow/tensionBoard 读工具，GM 调不到"以概念为单位"的读。
- **ARCH-004（承重层 port 未定 P2）**：DiceGm 直接耦合 Claude Agent SDK `query()` + Claude Code 专属 config（settingSources/allowedTools/mcpServers sdk type），`AgentFactory` 是 factory 非 port。
- **ARCH-006（终局机制 P1）**：game_end 工具只落 event + 标 session_meta.ended，触发条件完全靠 GM 自觉。
- **ARCH-007（rollGate 重启死锁 P2）**：waiters 内存态，后端重启后 DB 有 pending_roll 但 waiters 空，POST /roll 返回 409，fallback 未实现。
- **ARCH-009（prompt injection P1）**：玩家自由文本直接作 prompt 喂 GM，无隔离/标注。
- **ARCH-010（enrich 散落 P3）**：缝 A 出参贫信息补全在 DiceSession.enrich 而非 core。

我作为后端工程师，只验"实现是否落地了架构契约、调用链是否健壮、竞态/错误/可观测性是否到位、测试是否覆盖"，不验架构层设计自洽性（属架构）与玩家可见交互（属前端）。下面按"本领域漏洞→推导断节→给下游提示"展开。

  title: DiceGm 超时兜底是"脱困不恢复"——abort 后发 error 事件但游戏态停在半途，无重试无状态回滚，玩家看到错误后无路径继续

  severity: P1
  evidence: >
    `apps/orchestrator/src/dice/DiceGm.ts` L124-165：3min 超时（DICELORE_GM_TIMEOUT_MS 可覆盖）触发 `controller.abort()`，catch 里 `yield { type: "error", message }` 后回合结束，finally 清 timer + cleanupSkills。

  description: >
    架构层 [02 ARCH-007] 已标 rollGate 重启死锁是"gate 状态丢失"病根；本条是同源问题的另一面——超时兜底只解决"卡死"（进程脱困），不解决"半途状态"（数据一致性）。
    commit 30fdfb0 加超时兜底的直接动因是 eval 卡死，动机正确，但实现停在"脱困"层、未到"恢复"层：

  impact: >

    玩家场景：超时后静默失败（前端吞 error），玩家以为"没反应"再发消息，新回合基于半截污染状态跑——游戏态渐进腐坏。

  suggested_fix: >

    ② 中期：回合级事务——DiceGm runTurn 开始前记 `turn_start_seq = MAX(seq)`，超时/error 时 `DELETE FROM log WHERE seq > turn_start_seq` + 回滚 state 变更（需 core 提供回合级回滚原语，与 ARCH-002 快照机制同根）；
    ③ 长期：与 ARCH-002 快照接线合并——turnEnd 调 checkpoint()，超时后 restore 到本回合起点。

  hit_by: [后端]

## 推导断节

（上方 YAML 中 kind=推导断节 的条目按严重度重列聚焦：）

- **BE-003（P2）**：lore 路径设计了两路径共享流式 helper（streamDriverTurn）、LoreSession 刻意持 hub + attachWs/detachWs 方法，但 lore API 无 WS 升级端点、ws.ts 正则不匹配 lore 路径——LoreSession.hub.broadcast 的消息全发进空 hub 丢弃，attachWs/detachWs 是死代码。**设计意图（lore 构建流式）与实现（REST only）错位**。场景 C 团本构建黑盒，作者无实时反馈。
- **BE-007（P1）**：core game_end handler 是"GM 调即终局"的纯记录器，无框架侧触发条件校验（不查 sheet 钟/HP/回合数）——违 [00 立项前提]"AI 有讨好本能、不可信"；且 `GET /sessions/:id` 的 `ended: false` 硬编码不读 session_meta.ended，与 WS game_end 消息（B3 已修）信号矛盾——REST 报未终局、WS 报终局。**与 [02 ARCH-006] 架构层"何时敲/由谁敲"未定同根，后端侧 double-down：handler 无条件 + API 不读 meta**。

**共性病根（后端侧）**：后端实现存在三类系统性 gap——
1. **"脱困不恢复"模式**：超时（BE-001）、rollGate 重启（[02 ARCH-007]）都只解决"卡死"（进程脱困），不解决"半途状态"（数据一致性）——缺回合级事务/快照回滚兜底，与 [02 ARCH-002] 快照未接线同根。
2. **"路径不对称"模式**：dice 路径有 WS 流 + rollGate + turnEnd hook，lore 路径只有 REST + 无 gate + 无 hook——两路径共享底层（streamDriverTurn/LoreSession.hub）但 lore 侧接线缺失（BE-003），违 ADR-0023 ①"两路径共享底层"的对称性。
3. **"安全面分散"模式**：零鉴权（[02 ARCH-001/008]）、SSRF（BE-005）、团本注入（BE-006）、DELETE 越权（BE-004）是四个独立安全面，但病根同——"缝 B 零中间件 + 端点各自为政无统一校验"。发版闸前必须统一收口，非单点修补。
