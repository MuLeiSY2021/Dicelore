## 上游回放

接住 00-04 的关键锚定（聚焦各角色命中的 P0/P1 + 与"可验证性"强相关的条目）：

- **00 产品锚定**：
  - 验收口径＝相比裸 agent，AI 讨好本能被压下去、稳定不犯 F1/F2/F3（**相对判据非绝对分数**）；F1 掷骰绕过率（脚本可测，with 趋近 0）、F2 软着陆率（人评/裁判 agent）、F3 工具选择匹配率（人评）。
  - **PROD-002（P0）**：终局判据缺失——eval harness 跑多少轮都无法回答"做成了"（GM 不收局＝验收口径无法闭合）。
  - **PROD-003（P0）**：lore 侧（团本构建）零 eval——场景 C 的"作者↔构建 GM"交互质量、import 映射、构建工具可用性全无客观验证。
  - 发版闸＝承重绑定 ADR + port 契约、prompt injection 威胁建模、模型 key 托管+限流、多租户隔离（路线图第五批，里程碑三前必做）。
- **01 用户验真**：
  - **USER-008（P2）**：e2e 只验"开局→开场叙事"，掷骰→选择→终局整条玩局中后段零端到端验证；FAKE_GM 是脚本桩不触发 F1/F2/F3，真 GM（DiceGm）e2e 也没跑（烧 LLM）。
  - **USER-003/010（P1/P2）**：错误被 `.catch(()=>{})` 静默吞 + rollGate 内存态重启死锁——玩家可见故障路径无回归保护。
- **02 架构设计**：
  - **ARCH-002（P1）**：快照/回滚三层全断（snapshot 表未建、core 无 checkpoint/restore、Stop 不写快照）——即便补了 UI 也没快照可回滚，"超时后 restore 到回合起点"无机制兜底。
  - **ARCH-007（P2）**：rollGate 重启死锁——recovery.test.ts 测了重弹 roll_staged，未测"waiters 空 + POST /roll 409"死锁场景。
  - **ARCH-009（P1）**：prompt injection 威胁建模缺失——玩家自由文本直接进 GM 上下文，无注入测试。
  - 给测试提示：ARCH-002 快照 e2e、ARCH-007 重启恢复测试、ARCH-009 注入测试三类缺口。
- **03 前端落地**：
  - **FE-007（P2）**：e2e 只覆盖开局段（FAKE_GM 开场叙事），掷骰/选项/终局/错误恢复/断线重连全无 e2e；单测无错误路径与极端输入。
  - **FE-002（P1）**：错误吞没与错误通道不一致——postMessage/roll/choose 的 `.catch(()=>{})` 静默吞，HTTP 错误不进 error 通道。
- **04 后端落地**：
  - **BE-001（P1）**：DiceGm 超时兜底是"脱困不恢复"——abort 后已落库变更不回滚、已发 narration 不撤回、无重试。
  - **BE-002（P1）**：同一会话并发 handleMessage 无互斥——双击/重发/WS重连+REST 同时触发会双开回合，竞态无测试。
  - **BE-007（P1）**：game_end 触发完全靠 GM 自觉——core handler 无条件校验，`GET /sessions/:id` 的 `ended: false` 硬编码不读 session_meta.ended，REST/WS 信号矛盾。
  - **BE-009（P2）**：后端测试缺口系统盘点——无并发测试、无超时恢复测试、无注入测试、无 lore WS 测试、live 测试需烧 LLM 不进 CI。
