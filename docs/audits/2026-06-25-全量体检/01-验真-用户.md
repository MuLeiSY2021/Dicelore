# 用户 · 验真
> 上游文档：00-锚定-产品.md
> 本文档：作为终端玩家，基于产品锚定的"用户画像/核心场景/承诺做"逐一验真——我能不能顺畅玩一局、边界场景扛不扛得住、README 对我承诺的功能是否真能用。

## 上游回放

接住 00 的关键锚定：

- **用户画像**：两类直接用户——玩家（单人玩文字冒险游戏，诉求"尊重骰子、不崩坏、随时能玩"）+ 团本作者（声明式造团）；被塑形的 AI-GM 是塑形对象非用户。玩家需求四切面 A/B/C/D（[00 锚定单](00-锚定-产品.md) 用户画像表）。
- **核心场景 A（单人跑团，现在做）**：玩家 `dicelore play` 起 play 会话 → GM 经 MCP 读团本+塑形 skill 主持 → 每轮：玩家行动 → 塑形层判断给选项/掷骰 → resolver 产出叙述者无法伪造的结果 → narrate 散文 → 状态写入 sheet/event。完整使用闭环：开局→推进→掷骰→选择→状态推进→终局（game_end/you_death）。
- **承诺做（v1）**：行为塑形（L1/L2/L3 对抗 F1/F2/F3）、四业务域 SQLite+FTS5、跨端分发（npm 包 + `dicelore` CLI + 骑 Claude Code）。
- **承诺不做**：不取代人类 GM 上限、不造游戏引擎、单人阶段不追求玩家间碰撞、不商业化。
- **验收口径**：成功 = 相比裸 agent，AI 讨好本能被压下去、稳定不犯 F1/F2/F3（相对判据非绝对分数）。
- **PO 自扫的推导断节**：头号债 PROD-001（团本扩展零接线，场景 C 不可达）、PROD-002（终局判据缺失）、PROD-003（lore 侧零 eval）、PROD-004（D 流畅派无 backlog 承接）。

我作为终端玩家，只验"我能不能顺畅玩一局/用起来"，不验架构内部。下面按"核心场景能否跑通→体验断点→边界场景→README 兑现→横切越权"展开。

## 给下游的提示

**给架构师**：
- USER-006（零鉴权越权）是 [00 发版闸] "多租户隔离"的实证——当前 Hono app 零中间件，你侧评估时确认鉴权方案（session token / 用户登录态）与现有 `getOrCreateHost(id)` 路径的接入点。
- USER-010（rollGate 内存态重启死锁）的修复路径涉及 `handleRoll` 在 waiters 空时重驱 GM——这是 adapter/dice 层的状态机问题，需你侧确认"无 gate 立即掷"分支（debug 模式逻辑）能否安全复用到生产重连路径。
- USER-005（快照/回滚未接线）与 [00 PROD-002] 终局判据、ADR-0017 强耦合——你侧推 ADR-0017 接线时，须同步设计玩家侧回滚 UI 的契约（snapshot 表 → REST 端点 → PlayPage 按钮）。

**给实现**：
- USER-003（错误吞没）：PlayPage L122/L351 的 `.catch(()=>{})` 是低悬果，先改——让 error 进 useSession 的 error 通道统一显示。
- USER-002（终局锁死）：PlayPage 终局态加"重开/回滚/主页"三按钮，与 USER-005 回滚接线合并做。
- USER-007（API key 明文）：key 存 localStorage + 后端代发 SSRF——发版闸"模型 key 托管"已列，实现时优先收紧 /diagnostics/model-test 的 baseUrl 白名单（防 SSRF）。

**给测试**：
- USER-008（e2e 只覆盖开局段）：扩 FAKE_GM 脚本驱动掷骰→选择→终局闭环 e2e，是 dice 侧玩家路径回归的最低门槛——与 [00 PROD-003] lore 侧零 eval 对称补齐。
- USER-010（重启死锁）需专门的恢复测试：起局→掷骰等待→杀后端→重启→验玩家能否恢复。

**横切越权总结（USER-006/007）**：
- 当前实现下，orchestrator 一旦暴露到非本机，任意客户端可读/改/删任意 sessionId 的存档、可偷走玩家 API key、可借 /diagnostics/model-test 让后端代发请求到任意目标。
- 这是 [00 发版闸] "多租户隔离 + 模型 key 托管 + 限流"三项合一的 P0 风险面——发版闸前必须正面解，否则远程部署（AGPL 网络服务条款暗示的场景）直接致命。
- 本机单玩场景风险降级（无跨用户），但架构债仍在：sessionId 无主、key 明文、SSRF 面开着。
